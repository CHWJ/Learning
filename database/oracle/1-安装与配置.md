### 配置

- 配置监听器与远程连接 (Listener)
  - 默认情况下，Oracle 监听器可能只绑定了 localhost。如果需要远程连接：
  - 打开开始菜单，找到 Oracle - OraDb11g_home1 -> 配置和移植工具 -> Net Manager。
  - 展开 本地 -> 监听程序 -> LISTENER。
  - 在右侧的“地址”下拉菜单中，检查是否有主机的实际 IP 地址。如果没有，点击“添加地址”，输入服务器的本地固定 IP，端口默认 1521。
  - 保存配置。
  - Windows 防火墙设置：
      - 打开控制面板 -> Windows 防火墙 -> 高级设置。
      - 入站规则 -> 新建规则 -> 端口 -> TCP -> 特定本地端口输入 1521 -> 允许连接 -> 命名为 "Oracle Listener"。

- 游标数优化 (Open Cursors)
    > 高并发的 Java 应用（如使用了 Spring Boot / MyBatis 的连接池）容易报 ORA-01000: maximum open cursors exceeded（超出最大可打开的游标数）
    ``` sql
    -- 查看当前最大游标数（默认通常是 300）
    SHOW PARAMETER OPEN_CURSORS;

    -- 增大到 1000 或 2000（根据应用并发量调整）
    ALTER SYSTEM SET OPEN_CURSORS=1000 SCOPE=BOTH;
    ```

#### 密码策略

- 在 Oracle 11g 及更高的版本中，默认的 PASSWORD_LIFE_TIME 通常是 180 天
- 修改密码策略，让密码永久有效 `ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME UNLIMITED;`

#### Oracle 自动诊断仓（ADR）日志

``` bash
# 以管理员身份打开系统的命令行（CMD 或 PowerShell）
# 输入 adrci 进入交互界面

# 1. 切换到正确的 home
adrci> set home diag\rdbms\orcl\orcl

# 2. 再次执行清理（清理 7 天前的数据）
adrci> purge -age 10080 -type trace

adrci> show control
# 修改短周期的 trace 文件只保留 7 天
adrci> set control (SHORTP_POLICY = 168)

# 修改长周期的 incident 报警大包只保留 30 天（默认是 8760 小时，即 365 天，太长了）
adrci> set control (LONGP_POLICY = 720)
```
